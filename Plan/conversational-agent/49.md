# Run Full Test & Lint Suite Across Repos

## Scope
All services.

## Subtasks
- Run tests for `cursor-runner`, `jarek-va-ui`, and `elevenlabs-agent`.
- Fix any lint or type-check regressions from the new code.
- Ensure CI configuration includes elevenlabs-agent.

## Desired Outcomes

### Primary Outcomes
1. **Test Suite Success**: All tests pass across all services, ensuring no regressions.

2. **Code Quality**: All linting and type-checking passes, maintaining code quality standards.

3. **CI Integration**: New service integrated into CI pipeline for automated testing.

4. **Quality Assurance**: Confidence that all code meets quality standards before deployment.

### Success Criteria
- All tests pass
- No lint errors
- No type errors
- CI configured for elevenlabs-agent
- Test coverage acceptable
- Code quality maintained
- Ready for deployment

## User Stories

### As a Developer
- **Story 1**: I want all tests to pass so that I know nothing is broken.
- **Story 2**: I want CI integration so that tests run automatically.

### As a Code Reviewer
- **Story 3**: I want code quality checks so that I can verify code meets standards.

## Suggested Automated Tests

### CI Tests
1. **Test Suite Execution**:
   ```bash
   # Verify all test suites run
   npm test --workspaces
   ```

2. **Lint Checks**:
   ```bash
   # Verify linting passes
   npm run lint --workspaces
   ```

3. **Type Checks**:
   ```bash
   # Verify TypeScript compiles
   npm run type-check --workspaces
   ```

## Inherent Risks

### Medium Risk
1. **Test Failures**:
   - **Risk**: Tests fail due to regressions or flakiness
   - **Mitigation**: Fix failures, investigate flakiness, improve test stability

2. **CI Configuration Issues**:
   - **Risk**: CI not properly configured for new service
   - **Mitigation**: Test CI configuration, verify builds, document setup

### Low Risk
3. **Lint Rule Conflicts**:
   - **Risk**: Lint rules conflict with code style
   - **Mitigation**: Configure lint rules, update rules if needed, team agreement
