# Rename Conversations to "Note Taking History" in UI Only

## Scope
`jarek-va-ui`.

## Subtasks
- Update labels in `ConversationListView.tsx` from "Conversation History" to "Note Taking History`.
- Update headings and labels in `ConversationDetails.tsx` and `ConversationDetailView.tsx`.
- Adjust any references in `App.tsx` or routing components that show "Conversations" in titles.
- Update comments in `src/api/conversations.ts` and `src/types/index.ts` to clarify "note-taking" purpose.
- Manually verify that endpoints remain `/conversations/api/*` and that behavior is unchanged.
- Update or add component tests that assert the new labels while confirming that data loading behavior is unchanged.

## Desired Outcomes

### Primary Outcomes
1. **Clear Terminology**: Users see "Note Taking History" instead of "Conversation History" to distinguish cursor note-taking from agent conversations, reducing confusion about the two different systems.

2. **Zero Functional Changes**: All functionality remains identical - only UI labels and comments change. API endpoints, data structures, and behavior are completely unchanged.

3. **Improved Code Clarity**: Code comments and type names reflect the actual purpose (note-taking for cursor context) rather than generic "conversation" terminology.

4. **Test Coverage**: Automated tests verify that:
   - New labels appear correctly
   - All existing functionality still works
   - No regressions in data loading or display

### Success Criteria
- All UI labels updated from "Conversation" to "Note Taking History" or "Note Taking Session"
- No changes to API endpoints or data structures
- All existing tests pass
- New tests verify label changes
- Manual QA confirms no visual or functional regressions
- Code comments updated to reflect purpose

## User Stories

### As a User
- **Story 1**: I want to see "Note Taking History" instead of "Conversation History" so that I understand these are notes for cursor context, not chat conversations with an agent.

### As a Developer
- **Story 2**: I want code comments to clearly indicate "note-taking" purpose so that future developers understand the distinction from agent conversations.
- **Story 3**: I want tests to verify label changes so that I can be confident the rename didn't break anything.

### As a Product Owner
- **Story 4**: I want the UI to clearly distinguish note-taking from agent conversations so that users understand the two different features.

## Inherent Risks

### High Risk
1. **Incomplete Label Updates**:
   - **Risk**: Some labels missed, creating inconsistent UI
   - **Mitigation**: Systematic search/replace, code review, visual QA, automated label checks

2. **Functional Regressions**:
   - **Risk**: Accidental changes to functionality during rename
   - **Mitigation**: Comprehensive test suite, code review focusing on behavior, manual QA

### Medium Risk
3. **Test Failures from Label Changes**:
   - **Risk**: Tests fail due to updated text, requiring test updates
   - **Mitigation**: Update tests in same PR, use data-testid attributes for stable selectors

4. **User Confusion During Transition**:
   - **Risk**: Users familiar with "Conversation History" may be confused by new label
   - **Mitigation**: Clear messaging if needed, gradual rollout, user feedback

### Low Risk
5. **Comment Updates Missed**:
   - **Risk**: Some code comments not updated, causing future confusion
   - **Mitigation**: Code review, grep for old terminology, documentation review

6. **Translation/Localization Issues**:
   - **Risk**: If app is localized, translations may need updates
   - **Mitigation**: Check for i18n strings, update translation files if applicable
