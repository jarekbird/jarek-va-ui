# Add Styles for Voice and Agent UI Elements

## Scope
`jarek-va-ui`.

## Subtasks
- Extend `src/styles/App.css` for:
  - Voice control buttons.
  - Connection status indicators.
  - Mic activity animation.
  - Agent conversation layout tweaks.
- Validate styles in light and dark contexts (as applicable).

## Desired Outcomes

### Primary Outcomes
1. **Consistent Styling**: Voice and agent UI elements match the existing design system.

2. **Visual Feedback**: Clear visual indicators for connection status, mic activity, and agent mode.

3. **Accessibility**: Styles support accessibility requirements (contrast, focus states, etc.).

4. **Theme Support**: Styles work in both light and dark modes if applicable.

### Success Criteria
- Voice controls styled consistently
- Status indicators clear and visible
- Mic animation smooth and noticeable
- Layout tweaks improve UX
- Works in light/dark modes
- Accessible (contrast, focus)
- No style regressions
- Tests cover styling

## User Stories

### As a User
- **Story 1**: I want clear visual feedback so that I know when voice is active.
- **Story 2**: I want consistent styling so that the app feels cohesive.

### As a Developer
- **Story 3**: I want accessible styles so that all users can use the features.

## Inherent Risks

### Medium Risk
1. **Style Inconsistencies**:
   - **Risk**: Voice styles don't match design system
   - **Mitigation**: Use design tokens, follow style guide, visual QA

2. **Accessibility Issues**:
   - **Risk**: Poor contrast or missing focus states
   - **Mitigation**: Accessibility testing, contrast checking, keyboard navigation

### Low Risk
3. **Animation Performance**:
   - **Risk**: Animations causing performance issues
   - **Mitigation**: Optimize animations, use CSS transforms, performance testing
